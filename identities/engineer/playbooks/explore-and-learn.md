# Playbook - Explore & Learn

> 纯探索场景。没有明确目标，先摸索再说。

## 状态：草案 🟠

## 适用场景

- 接触一个全新的领域/技术
- 没有明确需求，先看看能做什么
- 原型试错、概念验证
- "玩一玩，看看会发生什么"

## 核心原则

- **自由度最大，约束最少**
- 允许走弯路，弯路本身也是收获
- 重点是记录发现，不是交付产出
- 每次探索结束时回顾：学到了什么、发现了什么

## 流程

1. **设定方向**（不需要精确目标）：大概想探索什么
2. **自由探索**：不限制技术选择、不限制代码质量
3. **随手记录**：有意思的发现随时记下来
4. **收尾回顾**：
   - 学到了什么 → `evolution/insights/`
   - 发现了什么假设值得验证 → `evolution/open-questions/`
   - 形成了什么可复用的方法 → `evolution/frameworks/`（待归属）

## 不需要

- 代码规范
- 测试
- Agent 协作
- 任何形式的审查

# Insights - 认知沉淀

> 每条 insight 按日期追加到 `YYYY-MM-DD.md` 文件中。格式三行，带生命状态。

## 格式

```markdown
## 简短标题
判断：一句话说清楚核心判断
状态：🌱 新鲜（未验证）
来源：平台 + 上下文
```

## 状态流转

```
🌱 新鲜（未验证）→ 🔥 验证中 → 🪨 沉淀
```

- **🌱 新鲜**：刚产生的判断，还没在实践中验证
- **🔥 验证中**：在后续对话/项目中遇到相关场景，正在验证
- **🪨 沉淀**：多次验证通过，可以考虑反哺到 Soul 或 Identity

AI 在后续对话里遇到相关场景时，主动更新状态。

## 示例

```markdown
## 接口设计宁可慢
判断：宁可慢也要把接口设计对，不接受事后重构的代价
状态：🌱 新鲜（未验证）
来源：Symbiosis 产品讨论

## 数字体应定位为跨平台内核
判断：不替代各平台的执行能力，只做跨平台认知汇聚和人格一致性
状态：🔥 验证中
来源：CodeBuddy 对话，对比五大平台后得出
```
